These endpoints proxy the local HL node for read-only state queries. Responses come from our local non-validator node — no rate limits beyond your plan tier, sub-20ms typical latency.

## Vaults

<ParamField path="GET /v2/vaults" type="endpoint">
  List every active vault on HyperLiquid. Returns an array of vault summaries (vault address, leader, TVL, PnL history).
</ParamField>

```bash
curl -H "x-api-key: YOUR_KEY" https://hyper.polynode.dev/v2/vaults
```

<ParamField path="GET /v2/vaults/leading" type="endpoint">
  Vaults a specific user leads. Pass `?user=0x...` with the leader's address.
</ParamField>

<ParamField path="GET /v2/user/vaults" type="endpoint">
  Vault equity positions for a specific user. Pass `?user=0x...`.
</ParamField>

## Delegator state

<ParamField path="GET /v2/user/delegator-history" type="endpoint">
  Historical delegate/undelegate/withdraw events for a user. Pass `?user=0x...`.
</ParamField>

<ParamField path="GET /v2/user/delegator-rewards" type="endpoint">
  Unclaimed and historical staking rewards for a user. Pass `?user=0x...`.
</ParamField>

<ParamField path="GET /v2/user/delegations" type="endpoint">
  Current active delegations for a user. Pass `?user=0x...`.
</ParamField>

Real response for a user with no delegations:

```json
[]
```

## TWAP summaries

<ParamField path="GET /v2/user/twap-summaries" type="endpoint">
  Aggregated TWAP order status for a user — pending, executed, and canceled TWAPs. Pass `?user=0x...`.
</ParamField>

<ParamField path="GET /v2/user/twap-fills" type="endpoint">
  Individual slice fills for each of a user's TWAP orders. Pass `?user=0x...`.
</ParamField>

## Validator state

<ParamField path="GET /v2/validator/l1-votes" type="endpoint">
  Current L1 governance votes from active validators. Returns an array of vote records.
</ParamField>

```bash
curl -H "x-api-key: YOUR_KEY" https://hyper.polynode.dev/v2/validator/l1-votes
```

Real response (no active votes right now):

```json
[]
```

## Spot deployment

<ParamField path="GET /v2/spot-deploy-state" type="endpoint">
  Current spot token deployment state for a deployer. Returns the deployer's active deployments plus the current gas auction parameters. Pass `?user=0x...` (any address, returns global auction state even if the user isn't deploying).
</ParamField>

Real response:

```json
{
  "states": [],
  "gasAuction": {
    "startTimeSeconds": 1776225600,
    "durationSeconds": 111600,
    "startGas": "500.0",
    "currentGas": "500.0",
    "endGas": null
  }
}
```

The `gasAuction` object describes HyperLiquid's Dutch auction for spot pair deployment — gas price decays linearly from `startGas` over `durationSeconds`. See [Gas Auctions research](/research/gas-auctions) for the full mechanism.

## Why query these through hypernode

- **Zero external rate limits.** We serve from our local HL non-validator node. You're not sharing a throttled public endpoint with anyone.
- **Sub-20ms latency** for most of these queries (the local node is on the same machine as the API).
- **One API key** works across every endpoint in this list, the snapshot endpoints, the batch queries, and the WebSocket stream.
