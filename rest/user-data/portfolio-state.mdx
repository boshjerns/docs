`/v2/user/portfolio-state` returns everything you need to display a trader's full account state in one request. Instead of making 4 separate calls to `clearinghouseState`, `spotClearinghouseState`, `openOrders`, and `userAbstraction`, the server fans them out in parallel against the local HL node and returns the merged response.

<Note>
Response time: **20-30ms total** (4 parallel local node calls). Saves you 3 round-trips vs making each call individually.
</Note>

## Single-user endpoint

```bash
curl -H "x-api-key: YOUR_KEY" \
  "https://hyper.polynode.dev/v2/user/portfolio-state?user=0x711d5ee8df7f0075f5393f7c9197248e961542b1"
```

Response shape:

<ParamField body="perp" type="object">
  Full clearinghouse state (perp positions, margin summary, cross margin summary, withdrawable, asset positions with funding and liquidation prices).
</ParamField>

<ParamField body="spot" type="object">
  Spot balances — all tokens the user holds in their spot account.
</ParamField>

<ParamField body="openOrders" type="array">
  Open resting orders across perp and spot markets. Each order includes `coin`, `limitPx`, `oid`, `side`, `sz`, `origSz`, `reduceOnly`, `timestamp`, and `triggerCondition` if applicable.
</ParamField>

<ParamField body="abstraction" type="string">
  Account abstraction mode — `"default"` or a custom abstraction identifier.
</ParamField>

Any field can be `null` if that sub-query returned empty or the underlying call failed.

## Batch portfolio-states (up to 2,500 wallets)

For copy-trading platforms, portfolio trackers, or multi-user dashboards, query up to **2,500 wallets** in a single POST:

```bash
curl -X POST \
  -H "x-api-key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "users": [
      "0x711d5ee8df7f0075f5393f7c9197248e961542b1",
      "0xacd89cFCB82Ae1f843467D56b58796bb928C9E1A"
    ]
  }' \
  https://hyper.polynode.dev/v2/batch/portfolio-states
```

The response is keyed by wallet address. Each value has the same shape as the single-user endpoint:

```json
{
  "0x711d5ee8df7f0075f5393f7c9197248e961542b1": {
    "perp": { "...": "..." },
    "spot": { "...": "..." },
    "openOrders": [],
    "abstraction": "default"
  },
  "0xacd89cFCB82Ae1f843467D56b58796bb928C9E1A": {
    "perp": null,
    "spot": null,
    "openOrders": null,
    "abstraction": null
  }
}
```

### Dry-run mode

Pass `"dry": true` in the request body to get a stub response without touching the local node. Useful for building/testing your parser.

```bash
curl -X POST \
  -H "x-api-key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"users":["0x711..."],"dry":true}' \
  https://hyper.polynode.dev/v2/batch/portfolio-states
```

Real dry response:

```json
{
  "0x711d5ee8df7f0075f5393f7c9197248e961542b1": {
    "_dry": true,
    "abstraction": null,
    "openOrders": [],
    "perp": {
      "_dry": true,
      "assetPositions": [],
      "marginSummary": {"accountValue": "0.0"}
    },
    "spot": {"_dry": true, "balances": []}
  }
}
```

### Limits

- Single-user `/v2/user/portfolio-state`: no hard limit, rate-limited by tier
- Batch `/v2/batch/portfolio-states`: max **2,500 wallets per request** (lower than the 10,000 cap on other batch endpoints because each wallet fans out to 4 sub-queries)
- Concurrency: semaphore-limited to 100 concurrent portfolio-states, which means 400 concurrent underlying node queries

## Why this is useful

- **Copy trading dashboards**: fetch the complete state of every wallet you track in one call
- **Leaderboards**: pair with `/v2/account-value-snapshot` to identify top traders, then fetch their full portfolio-state
- **Risk analytics**: pull the entire active set to compute aggregate leverage, open interest, or concentration metrics
- **User profile pages**: show positions, spot holdings, open orders, and abstraction mode without chaining 4 REST calls
