The hypernode REST API provides endpoints for trading, market data, and account management on HyperLiquid. No SDK required. Works from any language that can make HTTP requests.

**Base URL:** `https://hyper.polynode.dev/exchange`

**Interactive docs (Swagger UI):** [`/exchange/docs`](https://hyper.polynode.dev/exchange/docs)

## Network selection

All endpoints accept a `network` query parameter:

| Value | Description |
|-------|-------------|
| `mainnet` | HyperLiquid production (default) |
| `testnet` | HyperLiquid testnet |

```bash
curl https://hyper.polynode.dev/exchange/prices?network=testnet
```

## Authentication

Endpoints fall into three categories:

### Public endpoints (no auth)

Market data, user lookups, and discovery endpoints are fully public.

```bash
curl https://hyper.polynode.dev/exchange/prices
curl "https://hyper.polynode.dev/exchange/user/state?address=0x1234..."
```

### Authenticated endpoints (trading + account)

Endpoints that place orders or read your private account data require authentication. There are two options:

**Option 1: Register your wallet and use an API key**

Register once, then use the key for all future requests. Your private key is encrypted at rest using Fernet encryption, never logged, and never leaves the server.

```bash
# Register (one-time)
curl -X POST https://hyper.polynode.dev/exchange/register \
  -H "Content-Type: application/json" \
  -d '{"private_key": "0xYOUR_KEY", "network": "mainnet"}'
# Returns: {"api_key": "hn_abc123...", "address": "0x...", "network": "mainnet"}

# Use the key
curl https://hyper.polynode.dev/exchange/positions \
  -H "X-API-Key: hn_abc123..."
```

**Option 2: Pass your private key per-request**

Stateless. Nothing is stored.

```bash
curl -X POST https://hyper.polynode.dev/exchange/order \
  -H "X-HL-Private-Key: 0xYOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"coin": "BTC", "side": "buy", "size": 0.001, "price": 50000}'
```

See [Authentication](/rest/trading/authentication) for full details.

### Pre-signed passthrough

If you handle EIP-712 signing yourself, forward the signed payload directly:

```bash
curl -X POST https://hyper.polynode.dev/exchange/raw \
  -H "Content-Type: application/json" \
  -d '{"action": {...}, "nonce": 123, "signature": {...}}'
```

## Error format

All errors return JSON with an `error` field:

```json
{"error": "No mid price for FAKECOIN"}
```

| Status | Meaning |
|--------|---------|
| 400 | Bad request (invalid parameters, missing coin, etc.) |
| 401 | Missing or invalid API key |
| 404 | Resource not found |
