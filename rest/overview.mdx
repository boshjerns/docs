The hypernode REST API returns JSON snapshots of HyperLiquid state. All endpoints are read-only.

**Base URL:** `https://hyper.polynode.dev`

## Authentication

All REST endpoints accept an API key as a query parameter or bearer header:

```bash
# Query parameter
curl "https://hyper.polynode.dev/api/prices?key=pn_live_..."

# Or header
curl -H "Authorization: Bearer pn_live_..." https://hyper.polynode.dev/api/prices
```

## Response format

All endpoints return JSON with `Content-Type: application/json`. Errors return:

```json
{ "error": "missing 'asset' parameter" }
```

with the appropriate HTTP status code (400 for client errors, 502 for upstream errors).

## Rate limits

REST endpoints share API key tier limits. WebSocket subscribers receive updates pushed automatically and don't count against REST quotas. See [Authentication](/api/authentication) for tier details.
