title: REST API
description: Snapshot endpoints for prices, orderbook, funding, trades, candles, and orderbook attribution
---

The hypernode REST API returns JSON snapshots of HyperLiquid market state. All endpoints are read-only and require an API key in production.

**Base URL:** `https://hyper.polynode.dev/api`

## GET /api/prices

All mid prices for every tradeable asset.

**Request:**
```bash
curl https://hyper.polynode.dev/api/prices
```

**Response:**
```json
{
  "BTC": "71041.5",
  "ETH": "2180.15",
  "HYPE": "38.4785",
  "SOL": "82.2065"
}
```

Flat object mapping asset name to mid price string. Includes all 537 assets (perps + spot pairs + variants).

---

## GET /api/orderbook

L2 orderbook for a single asset, top 20 levels per side.

**Query parameters:**
- `asset` (required) — asset name like `BTC`, `ETH`, `HYPE`

**Request:**
```bash
curl "https://hyper.polynode.dev/api/orderbook?asset=BTC"
```

**Response:**
```json
{
  "asset": "BTC",
  "bids": [
    { "px": "71040.0", "sz": "1.23456", "n": 8 },
    { "px": "71039.0", "sz": "0.87654", "n": 5 }
  ],
  "asks": [
    { "px": "71041.0", "sz": "0.95432", "n": 6 },
    { "px": "71042.0", "sz": "1.11111", "n": 4 }
  ],
  "updated_at": 1775702972409
}
```

Each level: `px` (price), `sz` (aggregate size), `n` (resting orders at that level). Bids descending, asks ascending.

## GET /api/trades

Recent fills for a single asset.

**Query parameters:**
- `asset` (required) — asset name
- `limit` (optional) — max fills (default 100, max 1000)

**Request:**
```bash
curl "https://hyper.polynode.dev/api/trades?asset=BTC&limit=10"
```

**Response:**
```json
[
  {
    "coin": "BTC",
    "side": "B",
    "px": "71041.5",
    "sz": "0.0007",
    "time": 1775702972409,
    "hash": "0x426a997c3574d5a343e40438b7e64202017d0061d077f475e63344cef478af8d",
    "tid": 1099957769785331,
    "users": [
      "0x40cc85e70197c641fd5f38d10ac5c1924b17cc5c",
      "0x9d1eb5f9121e78faa5455d1b51cbf1a0c6e8340c"
    ]
  }
]
```

`side` is `B` for buy aggressor, `A` for sell aggressor. `users` contains both maker and taker addresses.

## GET /api/candles

OHLCV candles for a single asset and interval.

**Query parameters:**
- `asset` (required) — asset name
- `interval` (optional) — `1m`, `5m`, `15m`, `1h`, `4h`, `1d` (default `1m`)
- `limit` (optional) — max candles (default 500, max 1000)

**Request:**
```bash
curl "https://hyper.polynode.dev/api/candles?asset=BTC&interval=1m&limit=10"
```

**Response:**
```json
[
  {
    "open": 71040.0,
    "high": 71045.0,
    "low": 71038.5,
    "close": 71041.5,
    "volume": 12.34567,
    "timestamp": 1775702940000,
    "trades": 47
  }
]
```

`timestamp` is the start of the interval bucket in ms. `volume` is in base asset units. `trades` is the count of fills aggregated.

## GET /api/assets

Metadata for all 519 tradeable assets (229 perps + 290 spot pairs).

**Request:**
```bash
curl https://hyper.polynode.dev/api/assets
```

**Response:**
```json
[
  {
    "name": "BTC",
    "asset_id": 0,
    "sz_decimals": 5,
    "max_leverage": 40,
    "market_type": "perp"
  },
  {
    "name": "PURR/USDC",
    "asset_id": 10000,
    "sz_decimals": 0,
    "max_leverage": 0,
    "market_type": "spot"
  }
]
```

Asset IDs: `0+` for perps, `10000+` for spot pairs, `110000+` and higher for spot margin variants, `1900000+` for HIP-4 prediction outcomes.

## GET /api/l4book

L4 orderbook with individual orders attributed to specific maker addresses. Unique to hypernode.

**Query parameters:**
- `asset` (required) — asset name

**Request:**
```bash
curl "https://hyper.polynode.dev/api/l4book?asset=DOGE"
```

**Response:**
```json
{
  "asset": "DOGE",
  "bids": [
    {
      "px": "0.43021",
      "sz": "12340",
      "n": 4,
      "orders": [
        { "user": "0x9d705164...", "size": 5000, "oid": 375092088242 },
        { "user": "0xd071d6d6...", "size": 3000, "oid": 375092088243 }
      ],
      "anonymous_sz": "4340"
    }
  ],
  "asks": [],
  "coverage_pct": 100.0,
  "tracked_orders": 412,
  "l2_updated_at": 1775702972409
}
```

Each level shows the L2 total size (`sz`), individual orders attributed to specific addresses (`orders`), and the unattributed remainder (`anonymous_sz`).

`coverage_pct` is the percentage of the book where individual maker addresses are known.

## POST /api/info

Generic passthrough that accepts any HyperLiquid info request body. Use this for any query type not covered by the dedicated endpoints above.

**Request:**
```bash
curl -X POST https://hyper.polynode.dev/api/info \
  -H 'Content-Type: application/json' \
  -d '{"type": "userRateLimit", "user": "0x..."}'
```

The request body is forwarded as-is. The response is the JSON returned by HyperLiquid.

## Authentication

Production REST endpoints require an API key passed as a query parameter or header:

```bash
# Query string
curl "https://hyper.polynode.dev/api/prices?key=pn_live_..."

# Or header
curl -H "Authorization: Bearer pn_live_..." https://hyper.polynode.dev/api/prices
```

See [Authentication](/api/authentication) for details on obtaining keys.

## Rate limits

REST endpoints share API key tier limits. WebSocket subscribers ([WebSocket API](/api/connection)) receive updates pushed automatically and don't count against REST quotas.
